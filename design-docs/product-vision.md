# Product Vision: 4-6 Method Coffee App

## Goal
「4-6メソッド」を日常で再現しやすくする iOS アプリを目指す。

- 豆の量（g）と味の方向性（甘め / 普通 / 薄め）を指定すると、
  **6投目までの注湯量（g）と時間目安**を算出する。
- 豆とお湯の比率は `1:15` を基準に、`1:10` から `1:20` まで 1 刻みで調整できるようにする。
- 浅煎り / 中煎り / 深煎りの違いを考慮し、調整提案を出す。
- レシピと実際の抽出結果を記録し、次回に改善できるようにする。

## Target user
- ハンドドリップを普段行う人
- 4-6メソッドを始めたい / 安定させたい人
- 味の再現性を上げたい人

## MVP scope
1. **Brew Calculator**
   - 入力: 豆量(g), 比率(1:10...1:20), 味プリセット(甘め/普通/薄め), 焙煎度(浅/中/深)
   - 出力: 1投目〜6投目の注湯量(g)と時間目安
2. **Brew Note**
   - 抽出時のメモ（香り、酸味、苦味、総評）
   - 実測値（総湯量、抽出時間）
3. **History**
   - 過去レシピの一覧と再利用
4. **Local Data Persistence (SwiftData)**
   - 豆データの永続化（追加・一覧・編集）
   - 抽出ログの永続化（保存・一覧・削除・再利用）
5. **Bean Profile（豆プロファイル）**
   - 店名と購入日を必須で保存し、同じ豆の再現・振り返りをしやすくする
   - 産地/銘柄、焙煎日、URL、メモを任意で保存する
   - 抽出ログは豆プロファイルに紐づけて表示する
6. **Live Activity（ロック画面/ダイナミックアイランド）**
   - 抽出タイマー開始中に、ロック画面から「何投目 / 何g / 累積g / 次まで何秒」を確認できる
   - 一時停止時はカウントダウンを停止状態として表示する
   - 抽出リセットまたは抽出完了時にライブアクティビティを終了する

## UX principles
- 一目で分かる（数値とタイムラインを同画面で確認）
- 片手で操作できる（入力項目は最小限）
- 調整しやすい（味プリセット変更時に即時再計算）
- 再現しやすい（豆情報と抽出ログがアプリ再起動後も残る）
- 抽出中に画面を閉じても、進行状況をロック画面で素早く確認できる

## Home UI direction (2026-03)
- ホーム（プランナー）は「1画面で設定完了できる」カード型レイアウトを採用する。
- データモデルは既存の `BrewInput` / `BrewPlan` を維持し、見た目のみを更新する。
- 画面構成:
  1. 基本設定カード（豆量、比率表示、総湯量）
  2. 味わい調整カード（前半40%: 甘め/標準/薄め）
  3. 濃さ調整カード（後半60%: 軽め/標準/濃いめ）
  4. タイマー遷移CTA
- 基本設定カードの比率は `1:15` 初期表示とし、`1:10` から `1:20` を 1 刻みで直接変更できる。
- 主要カラーは以下5色をデザイントークン登録して使う:
  - `#AB5024`
  - `#5E260A`
  - `#F7844D`
  - `#015E57`
  - `#24ABA0`
- フォントはアプリ内で統一し、同一のフォント設計（rounded）を基準にする。

## Brew Assistant UI direction (2026-03)
- 抽出タイマー画面はホームと同じテーマトークンを使い、配色・角丸・陰影・フォントを統一する。
- データ要素は既存の `BrewPlan` / `BrewSessionModel` を維持し、UIレイアウトのみ更新する。
- 画面構成:
  1. 抽出ガイドヘッダー（戻る、タイトル、メニュー）
  2. 大型リング進捗（現在投数、経過時間、注湯量）
  3. 次ステップ/総湯量カード
  4. タイマー操作ボタン
  5. スケジュールカード
  6. 抽出メモと評価入力カード

## Beans UI direction (2026-03)
- 豆一覧はダークブラウン基調のカードレイアウトで、ホーム/タイマーと同じデザイントークンを使う。
- データ構造と表示項目は既存の Bean 情報（豆名・店名・購入日）を維持し、視覚デザインのみ更新する。
- 画面構成:
  1. ヘッダー（戻る、タイトル、追加ボタン）
  2. 豆カード一覧
  3. 空状態カード（豆未登録時）

## Brew Logs UI direction (2026-03)
- 履歴画面はテーマ統一されたカード一覧で表示し、ログ情報の可読性を優先する。
- 表示項目は既存の抽出ログ情報（豆名、日時、入力条件、抽出結果、メモ、評価）を維持する。
- 画面構成:
  1. ヘッダー（戻る、タイトル）
  2. 履歴カード一覧
  3. 空状態カード

## Settings UI direction (2026-03)
- 設定画面はフォーム主体からカード主体に変更し、他画面と同じ見た目ルールを適用する。
- 設定項目とデータ構造（重量単位、通知ヒント、バージョン情報）は維持する。
- 画面構成:
  1. ヘッダー（戻る、タイトル）
  2. 単位カード
  3. アシスタントカード
  4. 情報カード

## Success criteria (MVP)
- 豆データが再起動後も保持される。
- 抽出ログが再起動後も保持され、履歴から再利用できる。
- 保存処理の失敗時にアプリがクラッシュせず、UI state が破綻しない。
- 抽出タイマー開始時にライブアクティビティが開始し、表示項目が進行に追従する。

## Future ideas
- 豆ごとの推奨プロファイル学習
- 気温/湿度など環境要因の補正
- 抽出記録からおすすめレシピを提示
